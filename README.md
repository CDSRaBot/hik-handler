# hik-handler
CLI utility for diagnosing and managing ISAPI-compliant devices (Hikvision first).

# Назначение программы 

На основании внешних файлов с описанием "команд" (плагины в папке "modules")  сформировать запрос к устройству через http-запрос и получив ответ (xml) распарсить его для вывода на экран или файл.

# Назначение модулей программы
`hik-handler.py` - точка входа и инициализации настроек программы, обработка `config.toml`

`- orchestrator.py` - центральный модуль программы, точка сборки.  сам данные не обрабатывает, только управляет распределением и передачей данных между модулями, фактически - менеджер процессов, бизнес-логики.

`logger.py`  - модуль, управляющий настройками логирования процесса работы программы

`loader.py` - обработка внешних плагинов с командами. Плагины представляют собой xml-пакет, оформленный в соответствии с xsd-схемой. Лоадер отслеживает новые плагины, считывает наборы команд и обеспечивает доступ к ним из других модулей программы

`validator.py` - по запросу лоадера проводит валидацию структуры плагинов на соответствие xsd-схеме и обеспечивает кэширование скомпилированной схемы для повышения производительности при массовой обработке модулей.

`resolver.py` - обеспечивает парсинг аргументов при подготовке запроса к устройству

`settings.py` - обеспечивает чтение config.toml и настройку модулей программы

`security.py` - Защищенный контекст для хранения учетных данных и параметров сети устройства. Использует frozen=True для обеспечения неизменяемости (Immutability). После инициализации ни один компонент системы не сможет случайно или намеренно подменить IP (параметр host), логин, пароль или таймаут.

`session.py` - Модуль сетевого взаимодействия (Network Session). Реализует транспортный уровень ISAPI с поддержкой Digest-авторизации.

`cli_terminal.py`  - Класс реализации интерактивного терминала на базе prompt_toolkit. Обеспечивает взаимодействие пользователя с Оркестратором.

`module-schema.xsd` - схема для валидации плагинов
